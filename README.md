# linksee

## 项目简介
`linksee` 是一个基于 ROS 2 的移动机器人基础功能包，提供底盘串口控制、里程计发布、静态坐标变换以及二维激光雷达/建图相关启动能力。它主要解决机器人底盘控制链路打通、基础里程计接入和 SLAM 相关组件快速集成的问题，帮助开发者以较低成本完成从底盘驱动到感知建图的最小闭环搭建。

## 功能特性
- 支持通过 Launch 文件快速启动底盘控制、2D 激光雷达和 Cartographer 建图相关节点。
- 支持通过 Launch 参数调整串口、轮径、轮距、PID、前馈等控制参数。
- 支持小车激光雷达里程计启动。

## 快速开始
以下步骤用于帮助开发者以最短路径完成 `linksee` 包的构建与基础运行验证。

### 环境准备
- 已在K3上安装 ROS 2 运行环境，并完成基础环境变量初始化。
- 已具备 Python 依赖运行条件，包含 `rclpy`、`serial` 及相关 ROS 2 Python 运行时。
- 已接入底盘，并确认设备节点可用，例如 `/dev/rpmsg0`。
- 如需启动激光雷达或建图功能，请额外准备对应驱动与 `cartographer_ros` 相关依赖。

### 构建编译
在工作区根目录完成 ROS 2 工作区构建，确保 `linksee` 包被正确编译并安装。

### 运行示例
- 启动底盘控制与里程计节点：使用 `base_control_esos.launch.py`。
- 启动二维激光雷达：使用 `start_rplidar.launch.py` 或 `start_ydlidar.launch.py`。
- 启动激光雷达里程计：使用 `start_odom.launch.py`。

建议按“底盘控制节点 → 传感器节点 → 建图节点”的顺序逐步验证，并重点检查 `/cmd_vel`、`/odom`、TF 树及激光数据话题是否正常。

## 常见问题
- 底盘不运动：请检查是否正常发送 `cmd_vel`，以及底盘控制器串口协议是否与当前节点实现一致。
- `odom` 数据异常：请重点核对轮径 `wheel_diameter`、轮距 `wheel_base`、编码器参数、减速比及电机方向配置。
- TF 显示异常：请确认 `publish_tf`、`odom_frame`、`base_frame` 配置与系统其他模块保持一致。
- 建图效果不佳：请检查激光雷达安装位姿、静态 TF、里程计精度以及 Cartographer 配置文件是否与实际硬件匹配。
