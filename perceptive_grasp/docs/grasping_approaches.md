# 抓取方案

perceptive grasp 采用基于立体视觉的单目标顶抓方案。本页说明感知、定位、规划和执行之间的关系，以及底盘辅助的介入条件。

![perceptive grasp 系统链路](assets/grasp-system-flow.svg)

## 1. 感知目标

立体相机输出像素对齐的彩色图像和深度信息。yolov8-seg 在彩色图像中检测目标，并输出类别、检测框和分割掩码（mask）。

pipeline 优先统计分割掩码内靠近相机的稳健深度，减少背景或地面像素对目标距离的影响。目标区域缺少有效深度时，pipeline 停止当前任务，不使用背景深度继续规划。

realsense d435i 深度相机直接输出对齐深度。spacemit_las2 双目相机根据双目标定参数和视差推理生成与逻辑左目彩色图对齐的深度。两种后端向上层提供相同的彩色图、深度图和相机内参接口。

## 2. 定位与规划

系统使用立体相机内参将抓取像素反投影到相机坐标系，再使用 eye-to-hand 标定结果转换到机械臂基座坐标系。

抓取规划器根据目标掩码的主方向和长宽比计算夹爪方向。规划结果包含目标点、预抓取点、抓取点和夹爪 yaw。机械臂工作空间、关节约束和机身碰撞保护会在执行前限制可接受的规划结果。

## 3. 底盘辅助

底盘辅助属于抓取规划闭环，不属于机械臂执行阶段。目标位置超出 `mobile_base.target_x`、`x_tolerance` 和 `y_tolerance` 定义的舒适抓取区时，pipeline 进入 `BASE_ALIGNING`。

底盘每次只执行一个短距离前进、后退或原地转向动作。动作结束后底盘刹车，pipeline 等待画面稳定，再重新获取立体相机帧、检测目标并计算抓取位置。只有重新定位后的目标进入舒适抓取区，状态机才进入机械臂接近阶段。

pipeline 使用最大对齐次数、累计直行距离和视觉进展门限限制底盘动作。目标距离没有改善时，任务进入失败终态，避免持续移动。

## 4. 机械臂执行

执行器使用 pinocchio 求解逆运动学，并按接近、夹取、抬升、放置和归位的顺序控制机械臂。夹爪闭合后，系统结合夹爪位置和负载判断是否抓住目标；空抓或执行器错误会终止后续动作。

## 5. 方案边界

- 适用于桌面或近距离场景中的单目标顶抓。
- 依赖准确的立体相机标定、eye-to-hand 标定和目标表面深度。
- 不处理遮挡严重、透明、高反光或没有有效视差的目标。
- 底盘只执行局部辅助对齐，不提供导航、避障和全局路径规划。
- 抓取调试产物用于复盘单次任务，不替代碰撞检测或安全认证。
